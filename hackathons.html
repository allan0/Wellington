About the Hackathon

Join us for the flagship Python for AI Hackathon at Wellington Campus! This event is your platform to dive deep into the world of Artificial Intelligence using Python. Form teams, tackle real-world challenges across various tracks, and push the boundaries of innovation. Whether you're refining your skills or stepping into AI for the first time, this hackathon offers an unparalleled experience to learn, compete, network, and win exciting prizes.

Event Details

  • Date: 21st - 22nd April 2025
  • Time: See Detailed Agenda Below
  • Venue: Wellington Campus, Dubai Knowledge Park, Dubai, UAE
  • Format: Team-based (Teams of 2-4 recommended)
  • Theme: Python for AI Development

How to Get Ready

Ensure you and your team are prepared for success:

Hardware & Base Setup
  • Your own Laptop (Windows, macOS, or Linux)
  • Stable Internet Connection (Venue Wi-Fi available)
  • Charger & Power Adapter
Software & Tools
  • Python (3.8+ recommended) & Pip/Conda
  • Jupyter Notebook / JupyterLab
  • Code Editor (e.g., VS Code, PyCharm)
  • Git & GitHub Account (Essential for submission)
  • AI Dev Tools: Familiarity with Cursor (AI Code Editor) is a plus!
  • AI/ML Libraries: Experience via Hugging Face (Transformers, Datasets) or similar (Scikit-learn, TensorFlow/Keras, PyTorch)
  • Deployment: Account on Vercel, GitHub Pages, Streamlit Cloud, or similar platform.
Knowledge Prerequisites
  • Solid understanding of Python fundamentals (data types, functions, classes, modules).
  • Basic knowledge of AI/Machine Learning concepts (supervised/unsupervised learning, model training, APIs, core libraries).
  • Version control basics using Git (clone, add, commit, push, pull, branches).
  • Ability to work collaboratively in a fast-paced team environment.

Hackathon Process & Submission Checklist

Key steps and deliverables expected from each team:

Attend Briefing: Understand rules, tracks, and submission process (Day 1, 9:05 AM).
GitHub Repository: Create a public GitHub repo for your project code. Include a clear README.md.
Develop AI Solution: Build your core Python AI logic, following best practices.
Demo/Frontend (Recommended): Create a way to demonstrate your AI (e.g., Streamlit, Flask App, interactive notebook).
Deploy Demo (Recommended): Host your demo online (e.g., Vercel, GitHub Pages, Streamlit Cloud). Provide the live URL.
Prepare Pitch Deck: Create a short, compelling presentation (PDF format, max 5 slides: Problem, Solution, Tech, Impact, Team).
Submit Project (Day 1, by 2:30 PM): Submit through the official portal: GitHub repo link, live demo URL (if applicable), and pitch deck PDF.
Ensure all links and files are submitted before the deadline.
Deliver Pitch (Day 2): Present your project clearly and concisely during the allocated time slot.

Judging Criteria

Projects will be evaluated holistically by a panel of judges based on the following key criteria (weights may vary):

  1. Innovation & Creativity (25%): Originality of the idea, novelty of the approach within the chosen track, and creative use of AI.
  2. Technical Implementation (30%): Effective use of Python and relevant AI/ML libraries/tools, code quality, functionality, scalability potential, and technical difficulty overcome.
  3. Impact & Feasibility (20%): Significance of the problem addressed, potential real-world value/impact of the solution, and practicality of implementation.
  4. Execution & Demo (15%): Working demo quality (if applicable), completeness of the project within the timeframe, clear documentation (README).
  5. Presentation & Pitch (10%): Clarity, enthusiasm, structure of the presentation, ability to articulate the project's value, and Q&A responses.

Note: Compliance with rules (e.g., submission deadline, public GitHub repo) is mandatory.

AI-Assisted Evaluation

To complement the human judging process and provide objective insights, we may utilize AI tools to assist in the initial screening and evaluation. This helps ensure consistency and can include checks for:

  • Code Analysis: Basic checks for code quality, style consistency, and potential issues using static analysis tools.
  • Demo Accessibility: Verifying that the provided demo link is live and accessible.
  • Repository Checks: Basic verification of repository structure, README presence, and dependency file consistency.

Final scores and rankings are determined by the human judges based on the full criteria, integrating these AI-assisted insights where applicable. This is primarily for efficiency and preliminary feedback.
