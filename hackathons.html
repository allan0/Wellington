About the Hackathon

Join us for the flagship Python for AI Hackathon at Wellington Campus! This event is your platform to dive deep into the world of Artificial Intelligence using Python. Form teams, tackle real-world challenges across various tracks, and push the boundaries of innovation. Whether you're refining your skills or stepping into AI for the first time, this hackathon offers an unparalleled experience to learn, compete, network, and win exciting prizes.

Event Details

  • Date: April, 2025 (Final Date TBC)
  • Time: 10:00 AM - 6:00 PM (Full Day Event)
  • Venue: Wellington Campus, Dubai Knowledge Park, Dubai, UAE
  • Format: Team-based (Teams of 2-4 recommended)

How to Get Ready

Ensure you and your team are prepared for success:

Hardware & Base Setup
  • Your own Laptop (Windows, macOS, or Linux)
  • Stable Internet Connection
  • Charger & Power Adapter
Software & Tools
  • Python (3.8+ recommended) & Pip
  • Jupyter Notebook / JupyterLab
  • Code Editor (e.g., VS Code, PyCharm)
  • Git & GitHub Account (Essential for submission)
  • AI Dev Tools: Familiarity with Cursor (AI Code Editor) is a plus!
  • AI/ML Libraries: Experience with libraries via Hugging Face (Transformers, Datasets) or similar (e.g., Scikit-learn, TensorFlow/Keras, PyTorch)
  • Deployment: Account on Vercel, GitHub Pages, or similar platform for hosting demos.
Knowledge
  • Solid understanding of Python fundamentals.
  • Basic knowledge of AI/Machine Learning concepts (what is supervised/unsupervised learning, model training basics, APIs).
  • Version control basics using Git (commit, push, pull, branches).
  • Ability to work collaboratively in a team environment.

Hackathon Process & Submission Checklist

Follow these steps to successfully complete the hackathon:

10:00 AM: Registration & Welcome: Check-in, get your welcome kit.
10:30 AM: Intro & Track Selection: Briefing, tech overview, finalize your track.
11:00 AM: Hacking Begins: Ideate, plan, and start coding!
GitHub Repository: Create a public GitHub repo for your project.
Develop Backend: Build your core Python AI logic (Jupyter Notebook for exploration is fine, but structure your code).
Develop Frontend (Optional but Recommended): Create a simple UI (e.g., Streamlit, Flask, basic HTML/JS) to showcase your AI.
1:00 PM: Lunch Break: Refuel and network.
2:00 PM: Hacking Continues: Refine, debug, integrate.
Deploy Application: Host your demo (e.g., on Vercel, GitHub Pages). Provide the live URL.
Prepare Pitch Deck (PDF): Create a short presentation (max 5 slides) explaining your project, tech, and impact.
4:30 PM: Code Freeze & Submit Project: Final commit to GitHub. Submit links and Pitch Deck PDF.
Submit GitHub repo link, deployed app URL, and PDF pitch via the submission portal (link will be provided).
5:00 PM: Project Presentations & Judging: Short demos from each team.
6:00 PM: Awards Ceremony & Closing: Announce winners and celebrate!

  • 1st Place: $1,000 Cash Prize + Potential Internship Opportunity + Team Trophy
  • 2nd Place: $500 Cash Prize + Free Course Enrollment Vouchers
  • 3rd Place: Premium Tech Gadgets

  • Skill Enhancement: Hands-on experience with Python, AI, and modern development tools.

Judging Criteria

Projects will be evaluated based on the following criteria:

  • Innovation & Creativity: Originality of the idea and approach within the chosen track.
  • Technical Implementation: Effective use of Python, AI/ML techniques, code quality, and functionality.
  • Impact & Potential: Significance of the problem addressed and the solution's potential real-world value.
  • User Experience & Design (if applicable): Clarity and usability of the demo/frontend.
  • Deployment & Completeness: Working demo, code hosted on GitHub, project completion level.
  • Presentation & Pitch: Clarity, enthusiasm, and effectiveness of the team's presentation.

AI-Assisted Evaluation

To provide objective feedback, we utilize AI tools to assist in the initial evaluation process. This may include automated checks for:

  • Code quality & best practices (via linting/static analysis).
  • Functionality checks on the deployed application link.
  • Dependency checks and reproducibility from GitHub.

While human judges make the final decisions based on the full criteria, these AI insights help ensure a thorough and fair review. You might see an "AI Test Score Summary" as part of your feedback.
