Hackathon Support Structure

Wellington Campus is committed to providing comprehensive support throughout Day 1:

  • Challenge Briefing & Clarifications: Detailed printed/digital briefs. Faculty available for conceptual questions.
  • Mentor Assignment: One dedicated mentor (faculty/industry) per team to guide logic, data interpretation, and roadmapping.
  • On-Ground Help Desk: Central point for technical queries (Excel, Python, UI/UX) and solution structuring guidance.
  • Pitch Coaching Station: Mid-day support for refining pitch structure, value proposition, and presentation.
  • Resource Access: Digital folder with sample datasets, AI toolkits, slide templates. (LMS access for finalists).
  • Marketing Support: Capturing student engagement through photos and videos.

End of Day 1: Expected Outcomes

By the wrap-up at 2:00 PM, each team should aim to have:

  • A rough working model or defined logic (e.g., Excel structure, basic Python script, UI prototype/wireframe).
  • A preliminary 3–5 slide pitch deck outlining the problem, proposed solution, and approach.
  • Submitted these materials electronically for Wellington’s internal review ahead of Day 2 evaluations.

Venue & Logistics (DAFZ/CommerCity)

The following facilities are requested to ensure a smooth hackathon experience:

  • Dedicated breakout spaces or zones for one-on-one mentor sessions.
  • A display screen or projector available for potential team dry runs or presentations.
  • If feasible, a basic recording setup for capturing pitch rehearsals or sessions could be beneficial.

How to Get Ready

Ensure you and your team are prepared for success:

Hardware & Base Setup
  • Your own Laptop (Windows, macOS, or Linux)
  • Stable Internet Connection (Venue Wi-Fi available)
  • Charger & Power Adapter
Software & Tools
  • Python (3.8+ recommended) & Pip/Conda
  • Jupyter Notebook / JupyterLab Key Tool
  • Code Editor (e.g., VS Code, PyCharm)
  • Git & GitHub Account (Essential for submission)
  • AI Dev Tools: Familiarity with Cursor (AI Code Editor) is a plus!
  • AI/ML Libraries: Experience via Hugging Face (Transformers, Datasets) or similar (Scikit-learn, TensorFlow/Keras, PyTorch)
  • Deployment: Account on Vercel, GitHub Pages, Streamlit Cloud, or similar platform.
Knowledge Prerequisites
  • Solid understanding of Python fundamentals (data types, functions, classes, modules).
  • Basic knowledge of AI/Machine Learning concepts (supervised/unsupervised learning, model training, APIs, core libraries).
  • Understanding how to find, load, and prepare datasets for AI models.
  • Version control basics using Git (clone, add, commit, push, pull, branches).
  • Ability to work collaboratively in a fast-paced team environment.

Hackathon Process & Submission Checklist

Key steps and deliverables expected from each team. Track your progress using the button at the bottom right!

Attend Briefing: Understand rules, timeline, and judging criteria.
Setup GitHub Repository: Create a public repo for your project code & documentation. Add all team members as collaborators.
Acquire & Prepare Data: Find relevant datasets (cite sources!), clean, and preprocess them for your AI model (Jupyter notebook preferred for exploration).
Develop AI Solution: Implement the core AI/ML model(s) using Python. Use libraries like Hugging Face, Scikit-learn, TensorFlow/Keras, or PyTorch. Commit frequently!
Build Demo/Frontend (Optional but Recommended): Create a simple way to showcase your AI's functionality (e.g., using Streamlit, Flask/Django, or a well-documented Jupyter Notebook).
Deploy Demo (Recommended): Deploy your demo application to a platform like Vercel, Streamlit Cloud, or GitHub Pages for easy access by judges.
Prepare Pitch Deck: Create a concise presentation (max 5-7 slides) covering problem, solution, AI approach, demo, team, and next steps.
Submit Project: Submit the GitHub repository link, deployed demo link (if any), and pitch deck via the official submission form before the deadline. Ensure repo includes a README.md.
Deliver Pitch: Present your project clearly and concisely during the allocated time slot. Be ready for Q&A.

Judging Criteria

Projects will be evaluated holistically based on the following criteria (weights may vary slightly):

  1. Technical Implementation (30%): Quality of code, effective use of Python and AI libraries, complexity handled, working demo/prototype functionality.
  2. Innovation & Creativity (25%): Novelty of the idea or approach, originality in addressing the chosen track's problem.
  3. Impact & Feasibility (20%): Potential real-world impact of the solution, clarity of the problem being solved, feasibility of the proposed solution.
  4. Presentation & Pitch (15%): Clarity and conciseness of the presentation, effective demonstration of the project, quality of the pitch deck, Q&A handling.
  5. Completion & Polish (10%): How complete the project is within the hackathon timeframe, overall user experience/polish of the demo, documentation (README).

Note: Compliance with rules, effective use of Git/GitHub, and team collaboration (observed during mentoring) will also be considered implicitly.
